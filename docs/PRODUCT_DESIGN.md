# 产品设计文档 (Product Design Document)

**项目名称**: EcoNarrative Studio (生态叙事工作室)
**版本**: MVP (1.0)
**日期**: 2025-12-12

## 1. 产品概述
EcoNarrative Studio 是一个基于社会科学理论（如布伦纳生态系统理论、马克思历史唯物主义）的辅助写作与世界构建工具。不同于传统的“填词式”或“完全黑盒”的 AI 故事生成器，EcoNarrative 强调**结构的深度**。它通过引导用户定义社会系统的各个层级（从微观家庭到宏观意识形态），利用 AI 模拟这些层级之间的互动，从而生成具有逻辑深度、社会复杂性和历史厚重感的故事大纲与编年史。

## 2. 核心价值主张
*   **理论驱动的深度**: 引入学术模型作为故事骨架，解决 AI 生成故事往往“空洞”、“缺乏逻辑关联”或“背景单薄”的问题。
*   **结构化共创**: 用户不再是单纯的 Prompt 输入者，而是系统的架构师。用户定义实体，AI 演绎互动。
*   **可视化的历史演变**: 通过时间轴和河流图，将抽象的社会影响力变化可视化，帮助用户直观把握故事节奏。
*   **一体化工作流**: 从世界观设定 -> 实体构建 -> 历史编写 -> 章节生成，提供闭环的创作体验。

## 3. 目标用户
*   **小说家/编剧**: 需要构建庞大、严谨世界观的专业创作者。
*   **TRPG (跑团) 主持人**: 需要快速生成具有丰富背景和势力互动的战役设定的 DM。
*   **社会学/历史爱好者**: 希望通过模拟推演来观察社会理论互动的极客用户。

## 4. 设计原则
*   **学术与易用的平衡**: 虽然底层逻辑复杂，但 UI 采用现代化的卡片式和可视化设计，降低认知门槛。
*   **白描与客观**: 在历史生成上，坚持“去理论化”的白描风格，模拟真实史书的质感，留给读者解读空间。
*   **数据的所有权**: 强调本地体验与云端存档结合，用户对生成的每一个实体拥有完全的编辑权。
*   **原生多语言**: 从架构底层支持国际化 (i18n)，当前支持中文与英语无缝切换，服务全球创作者。

## 5. 信息架构 (IA)
应用采用**多视图 (Multi-View)** 架构，围绕同一个 `WorldModel` 数据核心展开：

1.  **全局控制层**: 顶部导航、同步/构建按钮、存档管理。
2.  **系统参与者视图 (Participants)**: 数据的录入与管理（CRUD）。
3.  **时间轴视图 (Timeline)**: 数据的可视化展示（关系与趋势）。
4.  **史书视图 (Chronicle)**: 数据的宏观文本化输出（Markdown 编辑器）。
5.  **故事引擎 (Story Engine)**: 数据的微观叙事输出（章节生成）。
6.  **设置与配置 (Settings)**: 全局 AI 参数、语言偏好与系统管理。
